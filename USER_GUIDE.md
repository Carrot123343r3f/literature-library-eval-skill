# 文献库审计 Skill 使用说明

`literature-library-eval` 用于评估一套文献库是否足以支持综述写作。它面向系统、范围、叙事、快速和伞式综述，并覆盖计算机、工程、医学、生命科学、化学、材料、物理、地学、数学统计等 STEM 领域。

## 它会做什么

- 检查已确认关键文献是否已在库中（A1）。
- 验证检索式能否找回已知相关文献（A2）。
- 估计候选集遗漏风险（A3），不会用库条数冒充覆盖率。
- 审计检索是否在已声明范围内趋于稳定、是否存在关键子方向缺口。
- 检查证据类型是否适合问题、版本/预印本/撤稿警示、检索新鲜度和元数据健康。
- 生成 Markdown、HTML 与 JSON 审计报告，以及可复跑的查询和决策记录。

## 首次调用

可直接说：

> 使用 literature-library-eval 审计我的 Zotero 文献库，判断它能否支持一篇范围综述。

首次运行时，AI 会一次性确认：研究问题、综述类型、纳入范围、时间/语言限制、文献库位置、自动化程度、可用数据库和输出格式。确认后会自行完成常规检索、调词、去重、筛选与报告，不会反复打断你。

## 数据源与权限

默认使用开放数据源，例如 OpenAlex、Crossref、Europe PMC 和 arXiv。若设备已配置合法的机构访问或 API 凭据，也可增强使用 Scopus、Web of Science、IEEE Xplore、Embase、CAS SciFinder 等来源。

订阅数据库不可访问时，流程会继续使用可用来源，并在报告中标注该限制；不会虚构访问结果。不要把 API key 写入提示词或报告，建议通过环境变量或已配置连接提供。

## 如何阅读报告

每项结论均含证据状态：

- **实测**：有稳定标识符、查询快照或其他可复跑证据。
- **估计**：依赖多源或统计假设，报告会说明区间与局限。
- **自动判定**：由自动筛选或分类得到，应结合置信度阅读。
- **待人工核验**：存在合理候选，但版本、相关性或匹配仍不确定。
- **不可评估**：缺少所需输入，不会被强行转为分数。

“趋于稳定”只表示在报告声明的来源、范围和日期下没有继续发现明显高置信度新增；它不等于全世界所有文献都已收齐。引用量、期刊等级和近期论文占比同样不是通用的质量分数。

## 安装与迁移

Claude 的用户级路径通常是 `~/.claude/skills/literature-library-eval`；Codex 的用户级路径通常是 `~/.codex/skills/literature-library-eval`。复制整个 skill 文件夹即可迁移到其他设备。若从 GitHub 获取，保留文件夹结构，并在重新打开 Claude/Codex 或新建任务后使用。

## 输入建议

最好的输入是含 DOI、PMID、arXiv ID、标题、作者、年份、摘要和来源字段的 Zotero 导出或 JSON。对于严谨综述，也请提供研究问题、明确纳入/排除标准、已有关键论文或权威综述，以及允许检索的数据库范围。
