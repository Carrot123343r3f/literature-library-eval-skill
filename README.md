# Literature Library Eval · 综述文献库质量评估

> 用于综述撰写前的文献库质量评估——以可量化的指标体系判断库是否满足要求。

一个 Claude Code skill，基于**六维指标体系**客观评估为撰写综述而构建的文献库，输出红黄绿判别、阻断项标注与定向改进建议。支持 **5 种综述类型 × 多领域适配**。

---

## 设计目的

文献库构建过程中，研究者面临的核心问题：

- 检索是否**充分**（有无遗漏关键文献）
- 库是否**满足**综述撰写要求（尚需补充什么）
- 检索是否**饱和**（继续检索能否获得新增量）

上述问题难以凭直觉判断。本 skill 采用**可量化指标 + 学术方法学依据**（PRISMA / Cochrane Handbook / 信息检索经典理论），提供客观评估。

---

## 核心能力

| 能力 | 说明 |
| --- | --- |
| **六维指标** | 覆盖度 Recall / 饱和度 GGR / 均衡 CV·Gini / 质量 h-core / 时效 Recency / 可用性 |
| **5 种综述类型** | 系统综述 / 范围综述 / 叙事综述 / 快速综述 / 伞式综述——判据按类型自动调整 |
| **多领域适配** | 计算机科学 / 医学 / 社会科学 / 人文 / 工程——各领域数据库、锚点、benchmark 自动套用 |
| **claim audit 报告** | 每个判断附 verdict（达标判定）+ evidence（计算依据）+ gap（差距量化）+ 阻断性（按类型）|
| **数据来源严谨化** | 全域采用 capture-recapture 估计、seminal 采用共引交集、关键词采用灵敏度调优 |
| **诚实标注** | 每个数值附来源、方法与局限；数据不可得时降级为区间估计，不伪造精确 |

## 快速开始

### 前提条件

- 已安装 **Claude Code**
- 文献库位于 **Zotero**（需配置 Zotero MCP）或可提供 **BibTeX/CSV** 文件

### 触发方式

以下任一方式：
- 自然语言：在 Claude Code 中表述"评估我的文献库""综述检索是否充分""Zotero 库是否满足综述撰写要求"等
- 命令：`/literature-library-eval`

skill 执行流程：探测 Zotero MCP → 确认综述类型 → 识别领域 → 计算六维指标 → 输出 claim audit 报告。

---

## 评估流程

```text
Step 0  确认数据源（探测 Zotero MCP / 请求 BibTeX）
Step 1  确定综述类型（说明 5 类特征 + 推断 + 与用户确认）
Step 2  识别领域 + 采集数据（CS=OpenAlex / 医学=PubMed / 社科=WoS）
Step 3  计算六维指标（调用 compute.py）
Step 4  标注红黄绿 + 阻断项（依据类型×指标判别矩阵）
Step 5  输出报告（claim audit + ASCII 可视化 + 改进路线图）
```

## 六维指标

| 维度 | 核心指标 | 回答的问题 |
| --- | --- | --- |
| **A 覆盖度** | Recall（全域/benchmark）、Seminal 命中 | 应收录的文献是否齐备 |
| **B 饱和度** | 滚雪球增益 GGR、重复率 DRR、边际新源收益 | 继续检索是否仍有新增量 |
| **C 均衡** | CV/Gini、来源多样性 Shannon、各方向 Recall | 各子方向覆盖是否均衡 |
| **D 质量** | h-core、Tier-1 占比 | 收录论文的学术影响力 |
| **E 时效** | Recency、前沿覆盖 | 是否覆盖最新研究进展 |
| **F 可用** | 摘要率、PDF 率、去重干净度 | 库数据的可用性与完整性 |

> 快速综述附加：时间窗覆盖 + 局限性声明。伞式综述附加：综述覆盖度 + AMSTAR-2 综述质量评估。

## 5 种综述类型与判别依据

| 类型 | 目的 | 必达维度 | 可放宽 |
| --- | --- | --- | --- |
| **系统综述** | 回答具体问题（穷尽、无偏、可重复） | A1全域、B、F3、E、M1完整PRISMA | — |
| **范围综述** | 绘制主题全貌（广覆盖、不评质量） | A1全域、B、C、M1 PRISMA-ScR | D、E |
| **叙事综述** | 梳理脉络与洞察（代表性、不穷尽） | A1benchmark、D、M1检索式透明 | A1全域、B、E |
| **快速综述** | 紧急时效优先（简化系统流程） | A1时间窗内、**E1时效必达**、局限声明 | A1全域（时间窗外）、B |
| **伞式综述** | 综述的综述（对象为已有综述） | 综述覆盖、AMSTAR-2、M1 | — |

## 局限性

- **全域分母不确定**：依赖关键词与数据库覆盖，采用区间估计而非单值（可通过 capture-recapture 精确化）
- **饱和仅为已用方法的饱和**：单一方法饱和不等于全域饱和（需多源交叉印证）
- **AMSTAR-2 需人工操作**：伞式综述的综述质量评分超出自动化范围
- **质量代理非质量本身**：h-core / Tier-1 为影响力代理指标，不等同于论文实际学术价值
- **阈值为经验值**：提供区间与依据，非绝对标准；边界值不作过度解读
