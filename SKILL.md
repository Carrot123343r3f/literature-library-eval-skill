# 工程综述文献库审计

将文献库准备度拆为可验证的覆盖证据、检索过程证据、范围结构、工程证据适用性、更新风险与库健康。不要生成单一总分，也不要把篇数、引用量、期刊等级或附件数量当作质量结论。

## 范围与触发

仅用于工程问题：研究对象、系统、方法、工况、性能指标、标准、数据集、基线或应用场景应能被明确界定。遇到纯基础学科或临床问题，说明超出此 skill 的适用范围；可只报告题录健康，不得套用工程充分性结论。

## 一次性确认后自主运行

首次调用只确认一次：研究问题和纳入边界、综述类型、时间/语言限制、库位置、自动策略、允许的数据源和输出格式。提供 `references/autonomous-run.md` 中的选项与推荐项。确认后自主检索、调词、去重、筛选和生成报告；仅在缺少必要输入、无法访问已指定来源，或新发现会实质改变范围时暂停。

读取 `references/autonomous-run.md`、`references/engineering-profiles.md`、`references/coverage-model.md`、`references/noncoverage-model.md`，以及与问题对应的检索说明。保存查询计划、来源快照、筛选/去重/版本决策、失败日志和证据状态。绝不要求用户在提示词中提交 API key；只使用现有 MCP、环境变量或用户已配置的合法连接。

## 执行流程

1. 规范化库，提取 DOI、PMID、PMCID、arXiv、OpenAlex 等稳定标识符；将版本族与重复记录分开处理。
2. 选择一个或多个工程 profile，并记录置信度、数据库覆盖差异和工程问题框架。
3. 生成每个来源的字段化检索式与 `query-plan.json`；记录日期、字段、过滤器和完整原文。
4. 运行开放源采集；存在合法订阅源时增强检索。来源失败时继续其他来源并记录失败。
5. 依纳入标准自动筛选，保留每条决策、理由和置信度；低置信度候选不进入核心集合。
6. 构建 A1 基准集、A2 gold set 和 A3 多源候选快照。稳定标识符是 A1/A2 的唯一自动匹配依据。
7. 对 gold-set 漏项逐项诊断术语、缩写、字段、来源或过滤条件；最小化修改检索式并保存每轮结果。
8. 按 A–F 证据模型审计并输出 `audit.md`、`audit.html`、`audit.json`、输入快照和运行日志。

## 判断规则

- **A1**：仅计算库对稳定标识符基准集的实测召回。
- **A2**：仅计算已执行检索式对稳定标识符 gold set 的灵敏度；零结果是实测 0，不是不可评估。
- **A3**：仅用多源、去重、明确纳入边界的快照报告下界或带假设的区间；单源 count 不是 Recall 分母。
- **B**：只有独立验证、两轮低新增率、路径完成和低边际收益共同满足时，才称“在声明范围内趋于稳定”。
- **C**：先建立工程 taxonomy；集中度只描述来源依赖，不能替代关键层缺口判断。
- **D**：评估工程证据是否匹配工况、基线、指标、标准和应用场景；被引与 venue 仅作上下文。
- **E**：优先报告每个来源最后成功检索日期与更新风险。
- **F**：分别报告元数据、去重、版本族、全文/开放链接、来源追溯与更正状态。

所有结论标为：实测、估计、自动初筛、待人工核验或不可评估。估计或自动初筛不能单独形成阻断结论。

## 资源

- `references/engineering-profiles.md`：工程领域路由、来源与问题框架。
- `references/coverage-model.md`：A1–A3 的定义、输入与边界。
- `references/noncoverage-model.md`：B–F 的证据模型。
- `references/autonomous-run.md`：确认表、停止规则与诚实边界。
- `references/keywords.md`：工程检索式构建与 gold-set 迭代。
- `scripts/collect_open_sources.py`：开放源查询快照采集器。
- `scripts/run_audit.py`：从规范化输入生成审计包。

先运行脚本的 `--help` 并验证输入结构；脚本只负责可重复计算，领域判别、相关性初筛和集合构建必须记录理由与置信度。
