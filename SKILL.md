# 综述文献库质量评估

> 客观评估"为写综述而建的文献库"是否胜任。支持 **5 种综述类型 × 多领域**。不做单一总分，用维度画像 + 短板原则。

## 配套资源

- **`compute.py`** — 指标计算脚本（Recall/CV·Gini/Shannon/h-core/Recency 等）
- **`example-report.md`** — 输出样板（3DGS 叙事实例）
- **`references/indicators.md`** — 六维 + 独特指标详解（定义/公式/阈值/数据源/局限/降级）
- **`references/coverage-model.md`** — 三层覆盖模型（基准集召回、检索策略灵敏度、候选集覆盖估计）
- **`references/noncoverage-model.md`** — B–F 的趋稳、结构、证据适用性、时效与库健康模型
- **`references/autonomous-run.md`** — 一次确认后的自主运行协议、停止规则与诚实边界
- **`references/stem-profiles.md`** — 全 STEM 领域 profile（数据库、问题框架、质量重点）
- **`references/review-types.md`** — 5 类型详解（目的/必达/放宽/独特/依据/判定逻辑）
- **`references/domains.md`** — 领域适配（CS/医学/社科/人文的数据库/锚点/benchmark）
- **`references/data-sources.md`** — 数据来源严谨化（全域/综述/seminal 的确定方法：capture-recapture/共引/多源三角）
- **`references/keywords.md`** — 关键词优化（PICO/概念矩阵/灵敏度调优/PRISMA-S 报告）
- **`scripts/collect_open_sources.py`** — OpenAlex、Crossref、Europe PMC、arXiv 查询快照采集器
- **`scripts/run_audit.py`** — 可复跑的 JSON、Markdown、HTML 审计报告生成器

## 自主模式（默认）

首次调用先完成一次性确认：研究问题与范围、综述类型、库位置、自动判别策略、数据源策略、输出格式。确认后，按 `references/autonomous-run.md` 完整执行，不再为常规检索、调词、筛选、去重或报告步骤要求用户确认。

自主模式必须：读取 `references/autonomous-run.md`、`references/stem-profiles.md`、`references/coverage-model.md` 与相应领域的关键词说明；保存查询计划、来源快照、自动决策及置信度；对无凭据的订阅数据库自动回退至开放源；从不在输出中暴露 API key 或令牌。

自动查询时，先生成 `query-plan.json`，再运行：

```text
python scripts/collect_open_sources.py --plan query-plan.json --out source-snapshot.json
python scripts/run_audit.py --library library.json --benchmark benchmark.json --gold gold.json --query-hits query-hits.json --out audit-output
```

这些脚本只计算可验证的部分。领域判别、相关性筛选、基准集构建和检索式迭代由执行此 skill 的 AI 按自主运行协议完成，并把决定依据写入审计日志。

## 评估流程

### Step 0 · 确认数据源

1. 探测 `zotero_*` MCP 可用？ → 有则直接读（配置在 `~/.claude.json`，不需用户说位置）
2. 无 MCP → 问用户给 BibTeX/CSV，或 Zotero userid+apikey（`compute.py --lib zotero`）
3. 报告标注数据源（可复现）

### Step 1 · 确定综述类型（详见 `references/review-types.md`）

**必须先和用户确认**（即使之前说过也复述）。类型决定哪些维度"必达（阻断项）"。

五种支持类型：**系统综述 / 范围综述 / 叙事综述 / 快速综述 / 伞式综述**（其他如批判性/元分析按最接近套用，详见 review-types.md）。

**AI 推断**（看领域+目的）→ 输出判断+理由 → 让用户确认。

> **核心逻辑**：六维指标**阈值所有类型通用**（测库的客观质量）；变的是**哪些必达**（类型决定某红指标是"阻断项"还是"非阻断"）。快速/伞式有独特指标（时间窗/综述覆盖+AMSTAR-2）。

### Step 2 · 识别领域 + 采集数据（详见 `references/domains.md`）

1. **识别领域**（从库 venue/标题/摘要）→ 告诉用户"推断为X领域，用Y数据库"，确认
2. **采数据**：
   - 库元数据（Zotero MCP / BibTeX）
   - 基准集与 gold set（领域权威综述、指南或专家确认的必纳入工作；优先 DOI/PMID/arXiv ID）
   - 候选集规模与重叠（领域数据库的查询快照；只作覆盖估计，不作 Recall 分母）
   - 被引数（算 h-core）

### Step 3 · 计算指标（详见 `references/indicators.md`）

六维 A–F（覆盖/检索趋稳/范围结构/证据适用性与影响力代理/时效/库健康）+ 快速/伞式独特指标。覆盖采用三层模型；B–F 按 `references/noncoverage-model.md` 审计。不能可靠计算时标为“不可评估”，不得以库规模、引用量或 venue 等级伪造质量结论。

### Step 4 · 标红黄绿 + 阻断项（按判别矩阵）

**阈值**：使用领域 profile 的默认值或用户确认的标准，并在报告中写明来源。A1/A4 的阈值仅适用于定义明确、可稳定匹配的基准集；候选集规模不能产生 Recall 阈值。其余经验阈值见 `references/indicators.md`，不得表述为跨领域硬标准。

**类型×指标判别矩阵**（明确每类型下每指标的标准——必达/放宽/不适用）：

| 指标（阈值） | 系统综述 | 范围综述 | 叙事综述 | 快速综述 | 伞式综述 |
| --- | --- | --- | --- | --- | --- |
| M1 流程完备 | 完整PRISMA ✓ | PRISMA-ScR ✓ | 检索式透明 ✓ | 简化PRISMA+局限声明 ✓ | 完整流程 ✓ |
| A1 基准集召回（profile 阈值） | ✓ 必达 | ✓ 必达 | ✓ 必达 | ✓ 必达 | — |
| A2 检索策略灵敏度 | 穷尽努力 ✓ | 广覆盖穷尽 ✓ | △ 推荐 | 时间窗内穷尽 ✓ | 综述覆盖 ✓ |
| A3 候选集覆盖估计 | △ 证据支持 | △ 证据支持 | — | △ 证据支持 | — |
| A4 核心工作命中（profile 阈值） | ✓ 必达 | ✓ 必达 | ✓ 必达 | ✓ 必达 | — |
| B1–B4 检索趋稳证据 | ✓ 必达 | ✓ 必达 | △ 推荐 | △ 简化+局限 | ✓ 必达 |
| C1 范围映射 | ✓ 必达 | ✓ 必达 | △ 推荐 | △ 推荐 | △ 推荐 |
| C2 来源依赖/集中度 | △ 推荐 | ✓ 关注 | △ 信息性 | △ 信息性 | △ 信息性 |
| C3 关键层缺口 | ✓ 必达 | ✓ 必达 | △ 推荐 | △ 推荐 | — |
| D1 证据适用性 | ✓ 必达 | △ 推荐 | ✓ 关注 | △ 推荐 | △ 推荐 |
| D2 影响力上下文 | △ 信息性 | △ 信息性 | △ 信息性 | △ 信息性 | — |
| D3 可信度警示/版本 | ✓ 必达 | ✓ 必达 | ✓ 关注 | ✓ 关注 | ✓ 必达 |
| D4 可复现性线索 | △ 推荐 | △ 推荐 | △ 信息性 | △ 推荐 | △ 推荐 |

> **图例**：✓ 必达 = 阻断项（红了不补不合格）｜△ 放宽/推荐 = 非阻断（红了可接受）｜— 不适用
>
> **覆盖特殊规则**：A1 是库对明确基准集的实测召回；A2 是检索式对 gold set 的灵敏度；A3 只能报告由多源并集、重叠或抽样得到的区间估计。数据库命中总数、库总条数及其比值不是 Recall。无明确基准集时，将 A1 标为“不可评估”，不能用单源 count 填补。
>
> 维度状态不能机械取“最差分数”：先检查输入证据状态，再按研究问题与领域 profile 判断。**报告的每个阻断结论都必须写入证据状态、适用范围与未验证内容**。

### Step 5 · 报告（借鉴 claim audit：每个判断都给证据+缺口）

输出四部分：

**① 指标审计表**（不只红黄绿，给 verdict + evidence + gap）：

| 维度 | 指标 | 标准 | 实际 | 证据状态 | Verdict | Evidence（怎么算的） | Gap（差距） | 阻断? | 建议 |
| --- | --- | --- | --- | --- | --- | --- | --- | --- | --- |
| A | A1 基准集召回 | profile >80% | 91% | 实测 | 🟢 达标 | 4 篇权威综述的 DOI 基准集；逐条 ID 匹配 | +11% | — | 保持 |
| F | F2 PDF | >70% | 43% | 🔴 未达 | 库 extra 字段含本地PDF计数 | -27% | **阻断项** | arXiv 替代 |

- **Verdict**：🟢达标 / 🟡边缘 / 🔴未达（比单纯红黄绿更明确）
- **Evidence**：实际值**怎么来的**（数据源+方法，可复现）
- **证据状态**：实测 / 估计 / 不可评估 / 待人工核验。估计值不得作为阻断项的唯一依据。
- **Gap**：标准−实际（量化差距）
- **阻断?**：按当前类型（阻断项/非阻断）

**② ASCII 可视化**：红黄绿条形 + 六维雷达（不生成 PNG）

**③ 改进路线图**：只列**阻断项** + 高优先短板（verdict/标准/实际/gap/动作）

**④ 综合描述**（报告末尾的一段学术定性总结——不是数字罗列，而是对库整体画像的凝练判断）：

写这段时遵循：
- **覆盖四要素**：① 库的整体定位（规模/方向/来源构成/时效）；② 核心优势（哪几维强、强在哪）；③ 主要短板（哪几维弱、是否阻断）；④ 结论性判断（胜任何种综述类型 / 若转变类型需补什么）
- **语言风格**：学术、凝练、客观。避免口语化（不用"够好""不太行"），用精准表述（"满足要求""存在差距""具备基础"）。表意清晰，不堆砌术语，不空洞。
- **有依据**：每句判断对应前面的指标（如"时效优势"对应 Recency 83%），不凭空。

示例框架（据实调整）：
> 本库覆盖 [领域] 的 [N] 个方向，共收录 [X] 篇文献，其中 [Y]% 来自 [顶级会议/期刊]，[Z]% 发表于近两年，具备 [时效/权威性] 优势。核心里程碑文献实现全覆盖（Seminal [S]%），benchmark 召回率达 [R]%，表明 [领域共识工作的齐备程度]。[主要短板维度] 相对薄弱（[指标值]），但对 [当前综述类型] [不构成阻断 / 需补充]。综合判断：本库 [满足 / 基本满足 / 尚需补强] [综述类型] 的撰写要求，[可立即开展 / 建议先补 XX]。

> 每个数字都附 evidence（数据源+方法）——这是 claim audit 式的严谨（借鉴 paper-self-review 的 verdict+evidence+missing 模式）。报告格式照 `example-report.md`。

## 关键词构建（验证检索策略灵敏度，详见 `references/keywords.md`）

锚定核心术语 → 同义词扩展 → 子方向分解 → 去噪 → Gold-standard 验证（10–20 seminal 反测，<100% 扩词）

## 原则

1. **不做总分**（维度画像 + 短板，加权加总掩盖短板）
2. **区分口径**（基准集召回可实测；检索式灵敏度只评价策略；候选集覆盖仅能估计）
3. **诚实标注**（实测/估计；seminal 主观性；AMSTAR-2 待手动评）
4. **判据按类型**（红指标标"阻断/非阻断"）
5. **承认局限**（候选集估计依赖关键词、数据库和去重；饱和只对已用方法；降级时标注不可评估）

## 常见误判（AI 必看，避免这些坑）

- **CV 高 ≠ 分类错**：领域天然不均（热门方向大、小方向小）会让 CV 高，这是真实分布，不是分类问题。先判"领域不均"还是"搜集偏科"。
- **候选集规模大 ≠ 漏了**：它可能来自宽关键词、不同纳入标准或未去重版本。只给区间，并以基准集召回与检索式灵敏度交叉验证。
- **h-core 低 ≠ 质量差**：按 arXiv 版查被引会低估（用正式版主记录）。标注口径。
- **PDF 低 ≠ 库不可用**：顶会正式版多付费，但 arXiv 预印本内容等同——先看是否有 arXiv 替代。
- **DRR 低 ≠ 一定没饱和**：单源 DRR 低可能因换源（B3 边际高）。多源交叉印证。
- **阈值边界不过度解读**：79% vs 80% 无实质差异，给区间更诚实。

## 最佳实践

- **先基准集、再检索式、后估计候选集**：先测库是否包含必纳入工作，再测策略是否命中 gold set，最后才给候选集覆盖区间。
- **多源三角验证**：关键数据（全域/被引/seminal）≥2 源交叉，单源标注局限。
- **给区间不给单值**：全域/综述数等不定分母，给区间 + 方法 + 局限。
- **每个数字附来源**：报告里每个值都标"数据源+方法+时间"（PRISMA-S 式可复现）。
- **类型先定再判**：先确认综述类型，再用对应判据（阻断/非阻断）——顺序反了会误判。
- **诚实 > 精确**：宁可标"估算/区间/局限"，不伪精确。
