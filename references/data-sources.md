# 数据来源严谨化（references）

> 指标的可信度取决于"分母/基准怎么来"。本文件给出全域/综述数/seminal/被引的**严谨确定方法**，引学术金标准（PRISMA/Cochrane）。

---

## 一、候选集范围怎么估计

“该主题所有相关文献”通常不可观测，不能把任何单库 count 直接作为 Recall 分母。以下方法只能估计在已声明范围内的候选集，结果必须作为区间或下界报告。

### 方法 1：捕获-重捕获（capture-recapture，推荐）

- **原理**：两个独立检索源（A、B），重叠部分（C）估总数：`N ≈ A×B/C`，带置信区间。
- **例**：OpenAlex 搜得 A=2487，PubMed 搜得 B=2100，重叠 C=1800 → N ≈ 2487×2100/1800 ≈ 2900（带区间）。
- **依据**：生态学种群估计法，信息检索领域用于检索评估（Buckland & Gey 1994）。
- **报告**：给全域**区间**（如 2700–3100）而非单值，更诚实。

### 方法 2：多源上限法（lower bound）

- 取多个数据库结果的**并集**（去重），作为全域下界（至少这么多）。
- 例：OpenAlex ∪ DBLP ∪ Semantic Scholar 去重 = 3200 → 全域 ≥3200。
- 简单但只给下界。

### 方法 3：灵敏度分析（区间）

- 用不同关键词组合跑全域 count，取**范围**（min–max）。
- 例：宽词"gaussian"=4400，窄词"gaussian splatting"=2487 → 全域约 2500–4400，报告区间。

### 单源查询（最低证据等级）

- OpenAlex `cites:{锚点}+search:{关键词}` 只能得到一个查询快照的候选数；它可能受锚点、关键词、索引覆盖和版本重复影响。
- **报告方式**：标注“单源候选集规模（估计）”；禁止称为全域 Recall、DCI 或覆盖率。
- **降级策略**：没有基准集时，A1 应标“不可评估”；没有多源数据时，A3 仅保留单源候选数和局限，不能给覆盖结论。

## 二、综述数量（伞式综述的 A1 分母）怎么确定

伞式综述的库内是"综述论文"，分母 = 该主题所有综述。

### 方法

1. **数据库类型过滤**：OpenAlex `filter=type:review&search:{关键词}`；PubMed `publication_type:Review`；WoS `DT=Review`
2. **多源并集**：OpenAlex + PubMed + WoS 的综述并集（去重）
3. **人工核验**：自动筛出的"review"可能含非综述（如 conference paper 标错），人工/标题过滤
4. **capture-recapture**（同上）：两源综述重叠估总数

### 注意

- 综述的"类型"字段在数据库可能不准（会议论文标 review、或反之）。**标题+摘要核验**必要（含"survey/review/systematic/scoping"等词）。
- 报告综述数也**给区间**（数据库过滤+人工核验的范围）。

## 三、Seminal 集合怎么确定（客观化）

当前 skill 用"主观挑 20 个"——弱。客观化方法：

### 方法 1：共引分析（推荐，客观）

- **原理**：取 ≥3 篇权威 survey 的参考文献，求**交集**（多篇 survey 共同引用的 = 领域公认核心）。
- **操作**：Semantic Scholar API 拉 survey refs → 求交集 → 交集≥阈值（如≥2 篇 survey 都引）= seminal。
- **依据**：文献计量学的共引分析（Small 1973；领域共识的代理）。
- **优势**：客观（不靠主观挑），可复现。

### 方法 2：高被引 top-N

- OpenAlex/Papers With Code 的 most-cited，取 top-N（如 top-50 或被引≥阈值）。
- **局限**：偏老（被引累积），新论文漏。

### 方法 3：专家共识（Delphi，重型）

- 多位专家独立列 seminal → 统计共识。严谨但需人力，AI 难自动。

### 推荐组合

- **默认**：共引交集（survey 共引≥2）+ 高被引 top-50 并集 → 候选 seminal 集。
- 报告标注方法（"共引+高被引，N篇"）。

## 四、被引数（h-core）的数据源

- **OpenAlex `cited_by_count`**：综合，但按 arXiv 版查低估（用正式版主记录查）
- **最佳**：按**标题**搜 OpenAlex 取主记录（合并版），拿合并被引
- **降级**：arXiv 版被引（低估），标注"h-core 为下界"

---

## 五、多源三角验证（所有数据的严谨化原则）

> 单源不可信。每个关键数据（全域/被引/seminal）尽量≥2 源交叉。

| 数据 | 主源 | 交叉源 |
| --- | --- | --- |
| 全域 count | OpenAlex | DBLP / PubMed / WoS（领域） |
| 被引 | OpenAlex | Semantic Scholar / Google Scholar |
| seminal | 共引交集 | 高被引 top + 领域 survey |
| 综述数 | OpenAlex type:review | PubMed/WoS + 人工核验 |

**报告规范**（引 PRISMA-S）：每个数据标注**来源+时间+局限**，可复现。

---

## 六、降级策略（数据不可得时）

| 情况 | 降级 |
| --- | --- |
| OpenAlex 限速 | 用 benchmark 口径（分母确定）；或本地已有数据估算+标注 |
| 多源不可得 | 单源 + 标注"单源估算，建议多源精确化" |
| seminal 共引不可得 | 高被引 top-N + 标注"非共引，主观性高" |
| 全域无法估 | 只报 benchmark Recall（不报全域），注明"全域不可估" |

> **诚实原则**：宁可给区间/标注局限，不伪精确。报告里每个数字都附来源+方法+局限。
