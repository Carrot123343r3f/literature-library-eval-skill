# A–F 平级工程文献库审计模型

六个维度平级呈现、独立判定；不得先以 A 的表现推导 B–F，也不得用任一维度掩盖另一维度的阻断问题。每个子项报告原始量、采用阈值、证据状态、verdict、定位原因与行动；报告不计算总分。

## A 覆盖验证：是否命中已知目标

A1 基准集召回、A2 检索式 gold-set 灵敏度、A3 完整多源候选下界、A4 关键工程工作/标准命中。详见 `coverage-model.md`。

## B 范围结构：是否覆盖工程问题的关键切面

| 子项 | 量化指标与默认判据 | 定位意义 |
| --- | --- | --- |
| B1 taxonomy 完整 | 预期工程层定义率 `100%` | 问题框架是否遗漏切面 |
| B2 关键层覆盖 | 关键层高置信度记录数 `≥1`；系统综述默认 `≥2` | 工况、方法、指标、应用的明确缺口 |
| B3 分类可靠性 | 分类置信度均值 `≥0.80`，未分类率 `≤10%` | 结构结论是否由不可靠分类驱动 |
| B4 结构集中度 | Top-source share、HHI、Shannon、CV/Gini 均报告；单源 `>80%` 警示 | 索引/研究生态依赖，不等同低质量 |
| B5 分层覆盖验证 | 有基准/Gold 的关键层逐层计算 A1/A2 | 小而关键层的检索盲点 |

## C 工程证据适用性：证据是否能回答工程问题

| 子项 | 量化指标与默认判据 | 定位意义 |
| --- | --- | --- |
| C1 证据类型 | 每种必需证据类型至少 1 条高置信度记录 | 缺现场、仿真、基准、标准等证据 |
| C2 工况/场景 | 核心集工况字段率 `≥80%` | 部署边界不可比 |
| C3 基线/指标 | 基线或性能指标字段率 `≥80%` | 性能主张无法对照 |
| C4 标准/数据/版本 | profile 要求时字段率 `≥80%` | 可比性、复现或合规风险 |
| C5 验证强度 | 实验/仿真/现场验证类型与数量分别报告 | 不能把单一验证类型外推 |
| C6 可信度警示 | 撤稿/勘误逐条处置；关键结论仅预印本为警示 | 版本和同行评议风险 |

## D 时效与技术演化：是否反映当前工程知识状态

| 子项 | 量化指标与默认判据 | 定位意义 |
| --- | --- | --- |
| D1 检索新鲜度 | 快速领域 `≤30`、常规 `≤90`、慢变 `≤180` 天 | 特定来源更新过期 |
| D2 关键来源窗口 | 已计划关键来源成功检索率 `100%` | 未访问/失败来源掩盖更新风险 |
| D3 前沿窗口覆盖 | 最近时间窗的 A2 或完整多源候选证据 | 新术语、新平台或新标准缺口 |
| D4 版本时效 | 关键预印本、正式版、在线优先、标准版本的复核率 `100%` | 使用过时或错误版本 |
| D5 时间分布 | 年份分布、中位发表年龄、近窗比例报告 | 描述演化，不作跨领域质量分 |

## E 学术影响与知识关联：库是否连接到领域知识网络

| 子项 | 量化指标与默认判据 | 定位意义 |
| --- | --- | --- |
| E1 引用数据可得 | 核心集有来源和日期的引用数据率 `≥80%` | 影响力上下文不可解释 |
| E2 影响力分布 | 年龄/领域归一化引用分位、h-core、长尾分布报告 | 只作背景，禁止等同质量 |
| E3 知识关联 | 引文前向/后向路径完成率 `100%`；共同引文/参考文献重叠报告 | 库与关键工作脱节 |
| E4 渠道与来源多样 | venue、来源、文献类型分布；单一渠道 `>80%` 警示 | 单点索引或传播偏差 |
| E5 权威锚点 | 领域标准、基准、综述或关键工作命中率单列 | 工程共识节点是否缺失 |

## F 过程规范、库健康与可复现性：审计能否被复跑与信任

| 子项 | 量化指标与默认判据 | 定位意义 |
| --- | --- | --- |
| F1 查询规范 | 计划查询中有原文、字段、日期、来源、过滤器的比例 `100%` | 检索不可复现 |
| F2 路径完成 | 计划数据库、前向、后向、profile 必需路径完成率 `100%` | 过程缺口 |
| F3 趋稳证据 | 连续两轮高置信度新增 `<2%`、新来源收益 `<5%`、独立验证无漏项 | 只能判断“趋稳可/不可证明” |
| F4 决策追溯 | 核心集有来源、批次、筛选和排除/纳入理由率 `≥95%` | 无法解释选择 |
| F5 元数据 | 标题/作者/年/来源/稳定 ID `≥95%`，摘要 `≥80%` | 导入或补全问题 |
| F6 去重与版本 | 未处置精确重复 `0`；模糊/版本候选均有决定 | 静默误删或重复计入 |
| F7 访问与复现线索 | 可访问率 `≥80%`；代码/数据/协议覆盖率报告 | 获取或复现准备度 |
| F8 更正检查 | 关键记录有更正检查日期/来源率 `100%` | 撤稿、勘误风险 |

F1–F8 是过程性、规范化、可追溯性和库健康的唯一归属。系统综述通常将 F1/F2/F4/F6/F8 设为阻断项。
