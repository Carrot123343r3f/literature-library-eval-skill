# B–F 工程审计细分标准

先采用 `engineering-standards.md` 的默认配置；研究问题、综述类型、法规/标准更新速度或用户标准可覆盖这些默认值。每个子项均输出 **通过 / 警示 / 未达 / 不可评估**、证据状态、定位原因与行动。只有“实测”的未达项可成为阻断项。

## B 检索趋稳度

| 子项 | 默认判据 | 未达时定位 |
| --- | --- | --- |
| B1 高置信度新增率 | 最近连续两轮均 `<2%`；核心集小于 50 篇时同时要求新增 `≤1` 篇 | 检索式/术语仍有增益，继续迭代 |
| B2 独立路径完成 | 计划数据库、后向、前向及 profile 必需路径完成率 `100%`；失败路径不能算完成 | 补跑遗漏路径或明确限制 |
| B3 新来源边际收益 | 已启用新来源中，最近两项均 `<5%` 高置信度新增 | 某来源仍有价值，不能称趋稳 |
| B4 过程可追溯 | 每轮均有查询、日期、来源、筛选和去重决策；缺任一项为警示 | 补齐运行日志 |
| B5 独立验证 | 未用于调词的验证集无稳定 ID 漏项 | 检索式对已知关键工作仍失败 |

只有 B1–B5 都通过时，结论才可为“在声明范围内趋于稳定”；任一不可评估时结论是“趋稳不可证明”，而不是“未趋稳”。

## C 范围结构与代表性

| 子项 | 默认判据 | 未达时定位 |
| --- | --- | --- |
| C1 taxonomy 完整 | 100% 预期工程层有定义、分类规则和边界 | taxonomy 本身不完整 |
| C2 关键层覆盖 | 每个标记为关键的层至少有 1 条高置信度证据；系统综述可配置为 `≥2` | 特定工况/方法/指标/应用缺口 |
| C3 分类置信度 | 每层已分类记录的平均置信度 `≥0.80`，未分类率 `≤10%` | 自动分类不可靠，需复核 |
| C4 来源依赖 | 单一来源份额 `>80%` 为警示；不自动判未达 | 可能索引偏差或来源单点故障 |
| C5 分层 A1/A2 | 关键层若有基准/Gold，应达到该层配置的 A1/A2 阈值 | 小而关键层的检索缺口 |

均匀度不是目标；CV/Gini/Shannon 只解释结构，不能替代 C1–C5。

## D 工程证据适用性

| 子项 | 默认判据 | 未达时定位 |
| --- | --- | --- |
| D1 必需证据类型 | 问题声明的每个证据类型至少有 1 条高置信度记录 | 缺现场试验、仿真、基准或标准等特定证据 |
| D2 工况/场景字段 | 核心集 `≥80%` 有工况或应用场景 | 无法比较真实部署条件 |
| D3 基线/指标字段 | 核心集 `≥80%` 有基线或性能指标 | 无法做性能对照 |
| D4 标准/数据/版本 | profile 要求时，核心集 `≥80%` 有对应标准、数据集或装置/软件版本 | 可比性和复现实验不足 |
| D5 可信度警示 | 撤稿/勘误必须逐条处置；关键结论仅依赖预印本为警示 | 版本或正式发表状态风险 |
| D6 可复现性线索 | 代码/数据/协议覆盖率只报告，不作质量分 | 需要全文或人工复现审查 |

引用量、venue 和 h-index 可提供背景，永不替代 D1–D6。

## E 时效与更新风险

| 子项 | 默认判据 | 未达时定位 |
| --- | --- | --- |
| E1 来源检索新鲜度 | 快速领域 `≤30` 天，常规工程 `≤90` 天，慢变标准/基础设施 `≤180` 天 | 特定来源过期或未执行 |
| E2 完整来源窗口 | 每个计划的关键来源均有成功日期；失败/未访问来源单列 | 更新结论不完整 |
| E3 前沿窗口覆盖 | 对最近窗口的 A2 或多源候选检查；无输入即不可评估 | 近期术语/来源未覆盖 |
| E4 版本时效 | 预印本—正式版、在线优先—卷期、标准版本关系已复核 | 关键记录版本过期 |

论文年份比例仅作背景；实际阈值由 profile 的发展速度和用户时间窗决定。

## F 库健康与可用性

| 子项 | 默认判据 | 未达时定位 |
| --- | --- | --- |
| F1 核心元数据 | 标题、作者、年份、来源、稳定 ID 各 `≥95%`；摘要 `≥80%` | 导入/补全问题 |
| F2 精确重复 | DOI/PMID 等稳定 ID 重复组应为 `0` 个未处置 | 去重决策缺失 |
| F3 模糊候选 | 标题—年份候选和版本族 100% 有“保留/合并/待复核”决定 | 不应静默合并 |
| F4 访问状态 | 核心集 `≥80%` 有本地附件或合法可访问链接；不适用时说明 | 获取策略不足，不等于研究质量差 |
| F5 追溯 | 核心集 `≥95%` 有来源、检索批次、筛选决定 | 无法复跑或解释纳入 |
| F6 更正检查 | 关键记录 100% 有检查日期和来源；发现警示必须处置 | 撤稿/勘误风险 |

F2/F3/F5/F6 在系统综述中通常是阻断项；其余类型按 `review-types.md` 和用户确认的标准决定。
