# 工程综述类型与阻断规则

| 类型 | 核心目标 | 不能缺少的证据 |
| --- | --- | --- |
| 系统综述 | 对工程问题作可复现、低偏倚的综合 | 明确边界、A1/A2、完整检索日志、B 未证明稳定时的限制、版本/去重与筛选追溯 |
| 范围综述 | 绘制工程问题的范围和证据图谱 | taxonomy、A1/A2 或其不可评估声明、关键层缺口、来源与筛选追溯 |
| 叙事综述 | 给出有代表性的工程脉络与论证 | 透明检索、代表性基准、工程适用性检查、限制声明 |
| 快速综述 | 在限定时间下提供及时的工程证据 | 明确时间窗、来源限制、最近成功检索日期、缩减方法与限制声明 |
| 伞式综述 | 综合已有工程综述 | 综述集合的 A1/A2、版本与重叠管理、对综述质量的人工或专门工具评估 |

“不可评估”本身不是失败；把缺失证据伪装成通过才是失败。估计和自动初筛不得单独作为阻断结论。系统综述中，无法证明趋稳时应阻断“检索充分”的宣称，而不必阻断透明地报告限制的工作。
