# 自主运行协议

在用户完成一次性确认后，完成以下流程，不再为常规步骤提问。只有缺少库文件、研究问题、可用数据源凭据或会造成范围变化的输入时，才暂停。

## 一次性确认

按 `references/confirmation-template.md` 的四批次模板执行。（批次一：研究问题/综述类型/库位置，必须逐项 → 批次二：策略/来源/格式，有默认可跳过 → 批次三：补充材料，可选但提示 → 批次四：合法性声明，告知不征求）

## 自动状态机

1. **规范化库**：导入题录，抽取 DOI/PMID/arXiv/OpenAlex ID，合并版本族，记录去重与字段缺失。
2. **领域 profile**：根据标题、摘要、venue 和研究问题，从 `engineering-profiles.md` 选择一个或多个 profile；记录置信度和所用来源。
3. **初始查询**：用问题框架和 profile 词表生成每个数据库的字段化查询；保存原文、日期、数据库和限制条件。
4. **开放检索与增强检索**：运行开放源；若凭据存在，运行 profile 允许的订阅源。任一来源失败时继续其他来源并写入失败日志。
5. **自动筛选**：以用户确认的纳入标准判别候选；保存每条自动决策、置信度和理由。低置信度条目不进入高置信度核心集，但保留在附录。
6. **构建 A1/A2 集合**：从权威综述、指南、共同引文和高相关候选生成基准集与 gold-set 候选；以稳定标识符去重、跨子方向分层。除非用户选择人工终审，否则按自主策略冻结集合并记录来源。
7. **查询迭代**：检查 gold set 漏项。逐项诊断为术语、缩写、字段、数据库或过滤条件问题；每轮仅做最小修改，重跑并记录召回、结果量和抽样精确率。满足停止规则或达到 10 轮后停止；达到上限时报告未解决漏项。
8. **B–F 评估**：按 `indicator-dictionary.md` 与 `engineering-standards.md` 计算检索趋稳、来源依赖、主题平衡、检索新鲜度、元数据/版本/访问/更正状态；将无法验证的项标为不可评估。
9. **报告**：输出 Markdown、HTML、JSON 评估包和可复跑输入快照。将每一结论标为实测、估计、自动判定或不可评估。

## 查询迭代停止规则

只有同时满足下列条件才将检索称为“在声明范围内趋于稳定”：独立验证集没有已知漏项；连续两轮的高置信度新增纳入量低于当前核心集的 2%；新增来源的高置信度相关率低于 5%；所有已启用的前向/后向引文路径已运行。否则报告“尚未证明稳定”。这些是运行规则，不是跨学科的质量阈值。

## 自动判别的诚实边界

自动判别不是事实裁决。报告必须分开显示：高置信度核心集、低置信度候选、未访问来源和失败查询。对撤稿、研究设计风险、版本是否等价等项目，自动标记供复核而不编造结论。
