# 文献库评估指标字典

> 本文档的权威定义源为 `schemas/indicator-registry.json`。如有不一致，以注册表为准。

六个维度平级，不计算总分。名称刻意区分：覆盖看”找回”，饱和看”还会不会继续长”，平衡看”是否偏向一边”，质量看”核心与渠道”，时效看”是否跟得上”，可用性看”能否实际写综述”。

**共 22 个子项（伞式综述 +3 → 25 个）。**

| 维度 | 子项与易读名称 | 含义与计算/核验 |
| --- | --- | --- |
| A 已知文献找回情况（Recall） | A1 已知必纳入文献找回率 | `库∩基准集/基准集`；仅稳定 ID 匹配。 |
| A 已知文献找回情况（Recall） | A2 检索式找回已知相关文献的能力 | `检索命中∩Gold set/Gold set`；零命中为实测 0。 |
| A 已知文献找回情况（Recall） | A3 多源去重后的候选下界 | 至少两个来源的检索快照按稳定 ID 去重后的并集——回答"至少有多少篇不重复文献存在"。不是 Recall，也不是"漏了多少"。 |
| B 检索是否趋于饱和（GGR/DRR） | B1 核心库增长率（GGR） | 每轮高置信新增数/该轮开始前核心库；最后两轮均 `<2%` 才通过。 |
| B 检索是否趋于饱和（GGR/DRR） | B2 新增路径发现率（DRR） | 新路径带来的此前未发现高置信文献/该路径候选；各路径 `<5%` 才通过。 |
| B 检索是否趋于饱和（GGR/DRR） | B3 饱和结论的过程证据 | 所有计划路径完成且独立验证未发现漏项；低 GGR/DRR 本身不够。 |
| C 主题与来源分布是否失衡 | C1 关键主题是否既覆盖又不过度偏斜 | 同时计算各预期主题的记录数、最大主题占比、CV、Gini、归一化 Shannon；无空主题，默认 Top topic `≤0.70`、CV `≤0.80`、Gini `≤0.50`、Shannon `≥0.55`。配置 `target_share` 时再算与目标分布的 TVD，默认 `≤0.25`。支持 `opposing_viewpoint` 标记——标注含对立/竞争观点的主题，C1 会附加定性检查提示 cherry-picking 风险。 |
| C 主题与来源分布是否失衡 | C2 检索来源是否过度集中或过度分散 | 统计最大来源占比、CV、Gini、归一化 Shannon。默认警示：Top share `>0.80`、CV `>1.00`、Gini `>0.60`、Shannon `<0.45` 或（来源至少 3 个时）`>0.95`。低 Shannon 是集中，高 Shannon 是异常分散，均要求解释。同时计算作者集中度（单一作者占比 >25% 时提示课题组偏倚风险）。 |
| C 主题与来源分布是否失衡 | C3 每个关键主题是否有独立来源支撑 | 输入 `topic_source_counts`，逐主题检查至少 2 个来源、单一来源占比 `≤0.80`；缺输入即不可评估。 |
| C 主题与来源分布是否失衡 | C4 观点偏斜度 | 先声明中心主张，再汇总支持、质疑、条件性/混合和未分类证据。默认分类覆盖 `≥50%`；对可争议主张，单方方向性证据 `≤90%`、反方 `≥3` 篇。未建主张、分类不足或证据单边均为 warning，触发反向术语/失败条件/反例补检；单边不等于错误，故不判 fail。 |
| D 库是否反映当前研究状态 | D1 检索来源是否足够新 | 各来源距最近成功检索的天数；快变领域 30 天、常规 90 天、基础设施 180 天。 |
| D 库是否反映当前研究状态 | D2 近年文献比例是否足够 | AI/软件/通信：近 3 年 `≥40%`；常规工程：近 5 年 `≥35%`；土木/能源基础设施/航天/交通：近 7 年 `≥30%`。可由 profile 覆盖。 |
| D 库是否反映当前研究状态 | D3 前沿主题是否专门检索和验证 | 对新术语、新平台或新标准保存独立检索、Gold set 或多源快照证据。 |
| D 库是否反映当前研究状态 | D4 预印本和正式版本是否已区分 | 核验预印本—正式版、会议版—期刊版、在线优先—刊出版；不重复计数。 |
| E 库是否包含足够学术影响 | E1 引用核心规模（h-core） | 按可得 `cited_by_count` 排序，最大 `h` 使至少 h 篇各被引至少 h 次；仅作背景信号，不是研究质量裁决。真正的研究质量评估应使用与研究设计匹配的批判性评价工具。 |
| E 库是否包含足够学术影响 | E2 领域 Tier-1 文献覆盖 | 用 profile 合法配置的 Tier-1 venue 映射计算命中数和占比；没有映射即 `not_assessable`。仅背景信号。 |
| F 文献是否真正可用于写综述 | F1 检索过程能否复跑 | 保存查询原文、字段、日期、来源、过滤器；目标完整率 100%。 |
| F 文献是否真正可用于写综述 | F2 摘要信息是否足够 | 有摘要记录数/总记录数，默认 `≥80%`。 |
| F 文献是否真正可用于写综述 | F3 PDF或开放全文是否可获得 | 有附件或开放链接记录数/总记录数，默认 `≥80%`。 |
| F 文献是否真正可用于写综述 | F4 重复和不同版本是否已处理 | DOI 精确重复应为 0；相似题名、预印本—正式版候选必须有决定。 |
| F 文献是否真正可用于写综述 | F5 纳入决定能否追溯 | 有来源谱系的核心记录比例，默认 `≥95%`；还应保存纳入/排除理由。 |
| F 文献是否真正可用于写综述 | F6 撤稿和更正是否已核查 | 关键记录应保存核验来源、日期和处置；没有专门核验则不可评估。 |

`pass`、`warning`、`fail` 与 `not_assessable` 分别表示满足、风险、明确不满足和证据不足。任何单项都不能掩盖另一维度的问题。

## 伞式综述专用子项（仅 review_type = "umbrella" 时启用）

以下三个子项仅当综述类型为伞式综述时出现在评估表中。非伞式综述不出现在报告中。

| 维度 | 子项与易读名称 | 含义与计算/核验 |
| --- | --- | --- |
| A 已知文献找回情况（Recall） | A4 库内综述论文占比 | 自动分类（title 含 "survey/review/comprehensive review/systematic review/meta-analysis/scoping review/umbrella review/综 述/进展/回顾/元分析" + 文献类型字段匹配）+ 人工抽样核验。`umb_library_type_purity` = 高置信综述论文数 / 库总记录数，默认 ≥ 0.90。一次研究占比 > 10% 即 risk。 |
| C 主题与来源分布是否失衡 | C5 纳入综述的覆盖分布 | 汇总纳入综述的子主题、方法学类型（系统 vs 叙事 vs 范围 vs 元分析 vs 其他）和检索窗口（起止年份、覆盖数据库）。若数据可得，计算 CCA (corrected covered area) 衡量综述间原始研究重叠：CCA = (N - r) / (rc - r)，其中 N=纳入原始研究的总频次（各综述原始研究数之和），r=去重后唯一原始研究数，c=纳入综述数。默认 `umbrella_overlap_cca_warning > 0.15` 提示需解释（引自 Pieper 等关于伞式综述重叠的文献）。子主题空白、方法学类型单一、检索窗口断层均需在报告中标注。 |
| F 文献是否真正可用于写综述 | F7 综述质量评估就绪度 | 逐综述记录全文获取状态（用于 AMSTAR-2/ROBIS）、已选定评估工具及其适用性、当前评估状态（已完成/待评估/无全文无法评估）。不代替实际质量评估——AMSTAR-2 的 16 项完整评分超出本 skill 自动范围。仅报告就绪度：在纳入综述中，具备全文可进行质量评估的比例。 |
