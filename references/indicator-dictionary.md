# 文献库审计指标字典

六个维度平级，不计算总分。名称刻意区分：覆盖看“找回”，饱和看“还会不会继续长”，平衡看“是否偏向一边”，质量看“核心与渠道”，时效看“是否跟得上”，可用性看“能否实际写综述”。

| 母项目 | 子项与易读名称 | 含义与计算/核验 |
| --- | --- | --- |
| A 已知文献找回情况（Recall） | A1 已知必纳入文献找回率 | `库∩基准集/基准集`；仅稳定 ID 匹配。 |
| A 已知文献找回情况（Recall） | A2 检索式找回已知相关文献的能力 | `检索命中∩Gold set/Gold set`；零命中为实测 0。 |
| A 已知文献找回情况（Recall） | A3 多来源检索到的最少唯一候选数 | 完整来源快照按稳定 ID 去重后的并集下界；不是 Recall。 |
| B 检索是否趋于饱和（GGR/DRR） | B1 核心库增长率（GGR） | 每轮高置信新增数/该轮开始前核心库；最后两轮均 `<2%` 才通过。 |
| B 检索是否趋于饱和（GGR/DRR） | B2 新增路径发现率（DRR） | 新路径带来的此前未发现高置信文献/该路径候选；各路径 `<5%` 才通过。 |
| B 检索是否趋于饱和（GGR/DRR） | B3 饱和结论的过程证据 | 所有计划路径完成且独立验证未发现漏项；低 GGR/DRR 本身不够。 |
| C 库的来源与主题是否平衡 | C1 关键主题层是否平衡覆盖 | taxonomy 中每个预期主题/方法/应用层至少有配置数量的高置信文献。 |
| C 库的来源与主题是否平衡 | C2 来源分布是否过度集中或过度分散 | 统计最大来源占比、CV、Gini、归一化 Shannon。默认警示：Top share `>0.80`、CV `>1.00`、Gini `>0.60`、Shannon `<0.45` 或（来源至少 3 个时）`>0.95`。低 Shannon 是集中，高 Shannon 是异常分散，均要求解释。 |
| D 库是否反映当前研究状态 | D1 检索来源是否足够新 | 各来源距最近成功检索的天数；快变领域 30 天、常规 90 天、基础设施 180 天。 |
| D 库是否反映当前研究状态 | D2 近年文献比例是否足够 | AI/软件/通信：近 3 年 `≥40%`；常规工程：近 5 年 `≥35%`；土木/能源基础设施/航天/交通：近 7 年 `≥30%`。可由 profile 覆盖。 |
| D 库是否反映当前研究状态 | D3 前沿主题是否专门检索和验证 | 对新术语、新平台或新标准保存独立检索、Gold set 或多源快照证据。 |
| D 库是否反映当前研究状态 | D4 预印本和正式版本是否已区分 | 核验预印本—正式版、会议版—期刊版、在线优先—刊出版；不重复计数。 |
| E 库是否包含足够的学术质量信号 | E1 引用核心规模（h-core） | 按可得 `cited_by_count` 排序，最大 `h` 使至少 h 篇各被引至少 h 次；只作背景，不是总分。 |
| E 库是否包含足够的学术质量信号 | E2 领域 Tier-1 文献覆盖 | 用 profile 合法配置的 Tier-1 venue 映射计算命中数和占比；没有映射即 `not_assessable`。 |
| F 文献是否真正可用于写综述 | F1 检索过程能否复跑 | 保存查询原文、字段、日期、来源、过滤器；目标完整率 100%。 |
| F 文献是否真正可用于写综述 | F2 摘要信息是否足够 | 有摘要记录数/总记录数，默认 `≥80%`。 |
| F 文献是否真正可用于写综述 | F3 PDF或开放全文是否可获得 | 有附件或开放链接记录数/总记录数，默认 `≥80%`。 |
| F 文献是否真正可用于写综述 | F4 重复和不同版本是否已处理 | DOI 精确重复应为 0；相似题名、预印本—正式版候选必须有决定。 |
| F 文献是否真正可用于写综述 | F5 纳入决定能否追溯 | 有来源谱系的核心记录比例，默认 `≥95%`；还应保存纳入/排除理由。 |
| F 文献是否真正可用于写综述 | F6 撤稿和更正是否已核查 | 关键记录应保存核验来源、日期和处置；没有专门核验则不可评估。 |

`pass`、`warning`、`fail` 与 `not_assessable` 分别表示满足、风险、明确不满足和证据不足。任何单项都不能掩盖另一维度的问题。
