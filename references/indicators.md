# 工程审计指标说明

本文件只定义可审计证据，不提供跨工程领域通用的“及格线”。阻断性由综述类型、研究问题和用户确认的边界决定。

## A 覆盖

- **A1 基准集召回**：库与稳定标识符基准集的逐条匹配；未提供基准集即不可评估。
- **A2 查询灵敏度**：已执行查询结果与稳定标识符 gold set 的逐条匹配；零命中是实测零。
- **A3 候选覆盖**：多源去重并集给出下界；只有满足独立性等假设时才给 capture–recapture 区间。详见 `coverage-model.md`。

## B–F

- **B**：新增率、独立路径、边际收益和可追溯性共同构成趋稳证据。
- **C**：按工程 taxonomy 检查工况、方法、性能指标、应用与时间窗口的关键缺口。
- **D**：检查证据是否适配工程问题；被引数、venue、代码链接仅为上下文或初筛线索。
- **E**：报告每个来源最后成功检索日期与前沿窗口风险。
- **F**：报告字段完整度、题录重复、版本族、全文/开放链接、来源追溯和更正状态。

CV、Gini、Shannon、引用分布、附件比例和被引数可作为描述性字段，但不能自动标红、判定质量或替代上述证据。
