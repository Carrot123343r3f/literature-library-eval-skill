# 六维评估模型（A–F，共 21 子项；伞式综述额外启用 A4/C4/F7，共 24 子项）

> 权威定义源为 `schemas/indicator-registry.json`。本文档为方法论阐述。如有数值不一致，以注册表为准。

六个维度平级，不合成总分。A 覆盖看"找回了多少已知工作"，B 饱和度看"检索还会不会继续长"，C 平衡看"主题与来源是否偏斜"，D 时效看"库是否跟得上当前研究状态"，E 学术影响看"引用核心与领域渠道覆盖"（仅背景信号），F 可用性看"文献能否实际用来写综述"。真正的研究质量评估应使用与研究设计匹配的批判性评价工具，h-core、Tier-1、来源集中度和预印本都只是特定性质的信号，不等同于综述质量。

**为什么不合成总分？** 一个对比案例：ScholarEval 框架（Moussa 等, 2025）用 8 维度加权平均分（4.2/5.0 → "Strong"）来评估已完成的论文/稿件——这是合理的，因为成品需要一个整体质量印象。但文献库准备度诊断的目标不同：一个库可能 F1（检索可复现性）= 0 但 A1（benchmark recall）= 0.95——加权平均 3.8 看起来"还行"，实际上检索完全不可复跑，是致命盲区。六维平级、不合成总分的设计，正是为了确保**任何一个致命短板都不会被其他维度的表现淹没**。这与 AMSTAR-2 不做综合评分的原理一致：识别弱项比贴总标签更有行动价值。

子项权威定义见 [indicator-dictionary.md](indicator-dictionary.md)，默认阈值见 [engineering-standards.md](engineering-standards.md)。

## A 覆盖（已知文献找回）

覆盖不是一个单一数字。按以下三层顺序报告，每层标注证据状态。

| 层级 | 回答的问题 | 指标 | 可接受证据 | 不可替代物 |
| --- | --- | --- | --- | --- |
| A1 基准集召回 | 当前库是否收录已确认的必纳入工作？ | `库 ∩ 基准集 / 基准集` | 稳定标识符逐条匹配 | 数据库命中总数 |
| A2 检索策略灵敏度 | 检索式能否找回已知相关工作？ | `检索结果 ∩ gold set / gold set` | 查询快照与逐条命中表（推荐：独立验证集） | 库的当前规模 |
| A3 候选集覆盖估计 | 在既定范围内，检索源可能遗漏多少候选？ | 下界或区间估计 | 多源去重并集、灵敏度分析；必要时 capture–recapture | 单数据库 count 的倒数或比值 |

### A 层必填字段

- 研究问题、纳入/排除标准、时间窗、语言和文献类型。
- 基准集/gold set 的来源、构建日期、稳定标识符与版本规则。
- 每个数据库的完整检索式、字段、限制条件、检索日期与结果快照。
- 去重和版本合并规则；不能自动判定的匹配项必须进入人工核验队列。

### A 层证据状态

- **实测**：保留了可复跑的输入与逐条匹配记录，**且 A2 的 gold set 为独立验证集**（未参与检索式迭代优化）。
- **估计**：由明确假设下的多源或抽样方法获得；必须给区间和假设。A2 的 gold set 与 dev set 重叠（无独立验证集）时降为 estimated。
- **待人工核验**：存在候选匹配或缺少版本确认。
- **不可评估**：没有满足定义所需的输入。不得用替代数字伪装为实测。

A2 证据状态分离基于 `references/search-strategy-protocol.md` 的 dev/validation set 分离原则——详见该文档第 2 节。

### Capture–recapture 的限制

仅在两个来源近似独立、对象边界稳定、去重可靠且交集可确认时使用。数据库索引和关键词高度相关时，这一假设通常不成立；此时只报告多源并集下界和查询灵敏度分析。

### A4 综述类型确认（仅伞式综述，review_type = "umbrella" 时启用）

伞式综述的库内文献应是已发表的综述论文（survey/review/meta-analysis），而非一次研究（primary study）。A4 检查库内综述论文占比（`umbrella_library_type_purity`）+ 类型判定方法（title 关键词匹配 / 文献类型字段 / 人工标注）+ 置信度标注。若一次研究占比 > 10%，说明建库策略可能偏向了原始研究检索，需在报告中明确标注为风险信号。

> **当 review_type ≠ "umbrella" 时，A4 不出现在报告中。**

## B 饱和度（检索是否趋于饱和）

以连续轮次的数据共同判断，单轮数字不足以构成"趋稳"结论。

| 子项 | 含义 | 计算/核验 |
| --- | --- | --- |
| B1 核心库增长率（GGR） | 每轮高置信新增数 / 该轮开始前核心库 | 最后两轮均 < `b_ggr_threshold`（默认 0.02）才通过 |
| B2 新增路径发现率（DRR） | 新路径带来的此前未发现高置信文献 / 该路径候选 | 各路径均 < `b_drr_threshold`（默认 0.05）才通过 |
| B3 饱和过程证据 | 所有计划路径完成且独立验证未发现漏项 | 低 GGR/DRR 本身不够 |

### 检索路径必须是真正的独立通道

B2 的"路径"指**真正独立的发现通道**，而非同一数据库的不同检索式变体。宽/中/窄检索式只是同一数据库中的查询变体，不能单独充当 B2 的独立路径。

五类独立路径（详见 `references/search-strategy-protocol.md` §3）：
1. 数据库布尔检索（`db_boolean`）
2. 后向引文追踪（`backward_citation`）——不依赖关键词
3. 前向引文追踪（`forward_citation`）——不依赖关键词
4. 相关文献网络（`related_articles`）——基于共引/耦合
5. 标准/指南/核心会议论文集（`standards_guidelines`）——基于来源类型

每条路径的 `new_high_confidence` 按路径声明顺序去重——先声明的先计数，不能因全局去重顺序让 DRR 失真。

### A2 停止 ≠ B 停止

A2 的停止（检索式已优化到足够好）与 B 的停止（不再发现新的纳入文献）是两套独立判据。即使检索式能找回所有已知文献（A2 达标），也不代表检索结果中不存在新的高相关文献。反之，GGR/DRR 很低也可能是因为检索式本身太窄。最终评估时，A2 与 B 同时满足各自停止条件才可声称检索充分（详见 `references/search-strategy-protocol.md` §6）。

### B 层证据要求

- 至少两轮 search round，每轮含 `core_before` 和 `included_high`。
- 每条检索路径的 `source_marginal_yields[]` 含 `candidates`、`screened_high_confidence`、`new_high_confidence`、`dedup_rule`、`yield`，供第三方独立复算。
- 独立验证不通过 → B3 判定为 `fail`，不能以"低 GGR"替代。

## C 平衡（主题与来源分布是否失衡）

先检查主题分布，再检查来源分布，最后检查两者的交叉依赖。每个参数衡量不同维度的失衡（互不替代）：

| 参数 | 衡量 | 典型阈值 | 解释 |
| --- | --- | --- | --- |
| Top-share | 单一支配度 | ≤0.70（主题）/ ≤0.80（来源） | 最大桶占比 |
| CV（变异系数） | 离差敏感度 | ≤0.80（主题）/ ≤1.00（来源） | 对极端值敏感 |
| Gini | 整体不平等 | ≤0.50（主题）/ ≤0.60（来源） | 0=均匀，1=完全集中 |
| Shannon Hn | 信息多样性 [0,1] | ≥0.55（主题）/ ≥0.45（来源） | 低=信息集中在少数桶；高=碎片化（仅说明性信号，不自动报警） |
| TVD（仅 C1） | 偏离预期目标分布 | ≤0.25 | 有 `target_share` 配置时生效 |
| Author Concentration | 作者/课题组集中度 | 单一作者 >25% 提示偏倚 | 仅诊断性信号——提示可能过度依赖单一研究群体，不等于质量缺陷 |

C3 额外要求每主题至少 2 个来源且单一来源占比 ≤0.80。

### 对立观点检测（Contrasting Viewpoints）

C1 的数值均衡不等于内容均衡。如果主题分类（taxonomy）中标注了 `opposing_viewpoint: true`，C1 的诊断语会附加提示：
- "即使数量均衡，也应检查是否同时覆盖了议题的正反证据，避免 cherry-picking（只收录支持假设的文献）。"
- 如果一个预期的对立主题完全没有文献（记录数为 0），C1 会标为 `empty_topic`，同时指出"对立视角缺失——收录文献可能偏向单一论点"。

此检测是定性补充，不影响 C1 的数值判定（pass/fail/warning）。

### C4 综述间覆盖分布（仅伞式综述，review_type = "umbrella" 时启用）

统计纳入综述的子主题分布（各综述覆盖的主题/子方向）、方法学类型分布（系统综述 vs 叙事综述 vs 范围综述 vs 其他）、以及检索窗口分布（各综述覆盖的检索起止年份和数据库）。用于判断：

- 子主题是否存在空白或过度集中（所有纳入综述都在同一子方向）
- 方法学类型是否单一（如全是叙事综述，缺少系统综述作为高证据强度锚点）
- 检索窗口是否互补（是否存在某些年份被所有综述共同覆盖，另一些年份无人覆盖）
- 按 CCA (corrected covered area) 衡量综述间在原始研究层面的重叠程度—— `umbrella_overlap_cca_warning > 0.15` 时提示需解释重叠原因

计算 CCA 需要纳入综述的原始研究引用列表（通过全文提取或题录匹配）；若无法获取则标注 `not_assessable` 并说明缺失输入。

> **当 review_type ≠ "umbrella" 时，C4 不出现在报告中。**

## D 时效（库是否反映当前研究状态）

| 子项 | 含义 | 计算/核验 |
| --- | --- | --- |
| D1 来源新鲜度 | 各来源距最近成功检索的天数 | AI/通信 30 天、常规 90 天、基础设施 180 天（profile 决定） |
| D2 近年文献比例 | 近 N 年文献占比 | AI/通信 3 年 ≥40%、常规 5 年 ≥35%、基础设施 7 年 ≥30% |
| D3 前沿覆盖 | 前沿主题是否专门检索和验证 | 新术语、新平台或新标准保存独立检索证据 |
| D4 版本区分 | 预印本与正式版本是否区分 | 预印本—正式版、会议版—期刊版关系已核验 |

D2 的年份字段完整率 <50% 时自动降级为 warning（防止分母掩盖缺失数据）；profile 支持可控 ID（`computer_ai` / `electronics_communications` / `mechanical_manufacturing` / `materials_chemical` / `biomedical_engineering` / `civil_infrastructure` / `energy_environment` / `aerospace_transportation`），也保留文本回退兼容。

## E 学术影响（仅背景信号）

| 子项 | 含义 | 计算/核验 |
| --- | --- | --- |
| E1 h-core | 按可得 `cited_by_count` 排序，最大 h 使至少 h 篇各被引至少 h 次 | 仅背景信号，不等于研究质量 |
| E2 Tier-1 覆盖 | profile 配置的 venue 映射命中数和占比 | 没有映射即 `not_assessable` |

h-core 对库规模、发表年限和学科引用习惯非常敏感，报告中应同时显示引用数据覆盖率。Tier-1 依赖期刊/会议名称精确匹配。**不设硬阈值**，仅作诊断背景。

## F 可用性（文献是否可写综述）

| 子项 | 含义 | 典型阈值 |
| --- | --- | --- |
| F1 检索可复跑 | 查询原文、字段、日期、来源、过滤器齐全 | 需要 run-log 文件 |
| F2 摘要覆盖率 | 有摘要记录数 / 总记录数 | ≥80%（叙事/范围）；≥85%（系统） |
| F3 全文获取率 | 有附件或开放链接任一可用的记录比例 | ≥60%（叙事/范围）；≥80%（系统） |
| F4 去重与版本 | DOI 精确重复=0，版本候选有决定 | 需要 dedup-log 文件 |
| F5 来源可追溯 | 有来源谱系的核心记录比例 | ≥85%（叙事/范围）；≥95%（系统） |
| F6 撤稿更正核查 | 关键记录核验来源、日期和处置 | 需专门来源核验 |

F3 使用"附件或开放链接任一可用"的联合比例而非相加，避免同一记录双渠道重复计数。F4 的版本决定需要 dedup-log 文件存在且可解析。

### F7 综述质量评估就绪度（仅伞式综述，review_type = "umbrella" 时启用）

伞式综述的核心步骤是对每篇纳入综述进行方法学质量评估（AMSTAR-2 或 ROBIS）。F7 检查这一步骤的前置条件是否满足：

- **全文就绪**：每篇纳入综述是否具备全文（F3 的伞式阈值 ≥ 0.85）
- **评估工具标注**：是否已选定质量评估工具（AMSTAR-2 / ROBIS / 其他）并注明适用性
- **评估状态**：已完成评估 / 待评估 / 无全文无法评估 —— 逐综述记录

AMSTAR-2 的完整 16 项评分超出本 skill 的自动范围，需人工或专用工具完成。F7 只报告**就绪度**（能否开始评估），不代替实际质量评估。

> **当 review_type ≠ "umbrella" 时，F7 不出现在报告中。**

### F4 附注：DOI/书目一致性（诊断提示，非独立子项）

F4 的去重（DOI 精确重复）只检查库内重复，不检查引用准确性。本 skill 不会自动校验每条记录的 DOI 是否能在 Crossref/PubMed 查到、author/year/title 是否与来源一致。**如果评估中使用了来自文献管理软件（Zotero、EndNote 等）自动抓取的元数据，建议人工抽查 10–20 条记录的 DOI 有效性和著录完整性**，以免 Garbage In, Garbage Out 扭曲 C/D/E 维的自动判定。

> 完整的引文准确性验证（如 DOI 批量校验、Retraction Watch 交叉比对）属于未来可做的 `review-manuscript-audit` skill 范围，本 skill 仅提供此项诊断提示。
