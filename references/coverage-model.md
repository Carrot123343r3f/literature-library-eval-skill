# 三层覆盖模型

覆盖不是一个单一数字。评估时按以下顺序报告，并为每项标注证据状态。

| 层级 | 回答的问题 | 指标 | 可接受证据 | 不可替代物 |
| --- | --- | --- | --- | --- |
| A1 基准集召回 | 当前库是否收录已确认的必纳入工作？ | `库 ∩ 基准集 / 基准集` | 稳定标识符逐条匹配 | 数据库命中总数 |
| A2 检索策略灵敏度 | 检索式能否找回已知相关工作？ | `检索结果 ∩ gold set / gold set` | 查询快照与逐条命中表 | 库的当前规模 |
| A3 候选集覆盖估计 | 在既定范围内，检索源可能遗漏多少候选？ | 下界或区间估计 | 多源去重并集、灵敏度分析；必要时 capture–recapture | 单数据库 count 的倒数或比值 |

## 必填审计字段

- 研究问题、纳入/排除标准、时间窗、语言和文献类型。
- 基准集/gold set 的来源、构建日期、稳定标识符与版本规则。
- 每个数据库的完整检索式、字段、限制条件、检索日期与结果快照。
- 去重和版本合并规则；不能自动判定的匹配项必须进入人工核验队列。

## 证据状态

- **实测**：保留了可复跑的输入与逐条匹配记录。
- **估计**：由明确假设下的多源或抽样方法获得；必须给区间和假设。
- **待人工核验**：存在候选匹配或缺少版本确认。
- **不可评估**：没有满足定义所需的输入。不得用替代数字伪装为实测。

## Capture–recapture 的限制

仅在两个来源近似独立、对象边界稳定、去重可靠且交集可确认时使用。数据库索引和关键词高度相关时，这一假设通常不成立；此时只报告多源并集下界和查询灵敏度分析。
