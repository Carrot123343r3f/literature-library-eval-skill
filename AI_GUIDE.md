# AI 执行说明

以 `SKILL.md` 为唯一流程入口。仅处理工程研究问题；纯基础学科或临床问题必须说明超出范围。

首次调用使用一次确认表：研究问题与边界、综述类型、时间/语言限制、库位置、自主策略、允许来源、输出格式。确认后不中断常规工作；只在缺少必要输入、指定来源不可访问或范围发生实质变化时暂停。

## 不可违反的规则

1. A1/A2 只匹配稳定标识符；标题相似仅进人工核验队列。
2. 空查询结果的 A2 是实测召回 0；没有执行查询才是不可评估。
3. A3 需要多源、去重、明确边界与假设。单源 count 不得叫覆盖率或 Recall。
4. F 中只有独立验证、两轮低新增、路径完成和低边际收益同时成立时，才能称检索趋稳。
5. B 的工程 taxonomy 必须含分类规则、置信度和未分类项；篇数均匀不是目标。
6. E 的被引、venue、知识路径与 C 的预印本、代码链接只提供上下文或警示，不是研究质量裁决。
7. 不把 API key、令牌、绝对本地路径、受限全文写入输出。

## B–F 标准的使用

先读取 `engineering-standards.md`，把默认值与 profile、综述类型和用户协议合并后写入 `context.standards`。逐项执行 A1–A4、B1–B5、C1–C6、D1–D5、E1–E5、F1–F8；每项必须同时输出采用阈值、证据状态、verdict、定位原因和行动。不可自动验证的子项写 `not_assessable`，不得省略或默认为通过。

## 交付物

输出目录必须有：`audit.md`、`audit.html`、`audit.json`、`query-plan.json`、`source-snapshot.json`、`decision-log.json`、`deduplication-log.json` 和 `run-log.json`。对缺失来源、失败查询、低置信度候选和无法验证项单独列出。

脚本只计算规范化输入；调用者必须在运行前生成并保存上述日志。对研究设计有效性、撤稿状态、版本等价性与全文可复现性，只能输出自动初筛或待人工核验。
