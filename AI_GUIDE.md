# AI 执行说明

此文件面向调用 `literature-library-eval` 的 AI。执行时以 `SKILL.md` 为主流程；需要细节时按需读取 `references/`，不要把本指南当作用户可见报告。

## 运行契约

首次调用只进行一次性确认：研究问题与范围、综述类型、文献库位置、自动判别策略、数据源策略和输出格式。确认后自主运行；仅在缺少必要输入、无法访问用户指定数据源，或新发现会实质改变范围时暂停。

默认采用“全自动 + 混合增强”：开放源始终可用；仅在合法凭据或现有连接存在时启用订阅源。绝不把密钥、令牌、完整本地路径或未获授权的全文内容写进报告。

## 必须遵守的判断规则

1. A1 只能以 DOI、PMID、PMCID、arXiv ID 等稳定标识符匹配的基准集计算。
2. A2 评价检索式对 gold set 的灵敏度；不能以库规模替代。
3. A3 只能报告多源去重、灵敏度分析或满足假设的 capture–recapture 得出的估计/下界；单库 count 不是 Recall 分母。
4. B 不得以单一 GGR/DRR 宣称饱和。遵循 `noncoverage-model.md` 的趋稳证据规则。
5. C 的集中度和均匀度只描述结构；先建立研究问题驱动的 taxonomy，再判断关键层缺口。
6. D 的引用量、venue 和 h-index 仅为影响力代理，不能等同研究质量。明确区分证据适用性、可信度警示和可复现性线索。
7. E 优先报告检索新鲜度；出版年份必须按领域 profile 解释。
8. F 分别处理题录重复、版本族、附件/开放链接、来源追溯与更正状态。
9. 对每个结论标为：实测、估计、自动判定、待人工核验或不可评估。

## 自主执行顺序

1. 规范化输入，抽取稳定标识符，记录去重与版本候选。
2. 选择 `stem-profiles.md` 中一个或多个领域 profile。
3. 根据问题框架生成字段化检索式和 `query-plan.json`。
4. 运行开放源采集器；检测到已配置的合法订阅源时进行增强检索；记录成功、失败和日期。
5. 依照纳入标准自动筛选，保留条目级理由与置信度。
6. 从权威综述、指南、共同引文和高相关候选中构建 A1/A2 候选集合；分层并保留来源。
7. 对 gold set 漏项逐项诊断并最小化修改检索式，保存每一轮的召回、结果量、抽样精确率与新增纳入量。
8. 完成 A–F 审计，使用 `scripts/run_audit.py` 输出 JSON、Markdown 和 HTML，再补充有证据支撑的定性解释。

## 输出要求

输出目录至少包含：`audit.md`、`audit.html`、`audit.json`、查询计划、来源快照、筛选/去重日志和限制说明。报告应说明适用范围、未访问来源、失败查询、低置信度候选及无法验证的项目。

不要编造数据库访问结果、研究设计质量、撤稿状态、全文可得性或“检索穷尽”结论。
